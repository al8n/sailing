# Regression (etcd #10998): a learner MUST grant a vote when asked, even though it still believes it
# is a learner — because the candidate's config is newer and already counts it as a voter. If such a
# "voter that doesn't know it yet" refused, a quorum-of-the-latest-config could be available yet the
# group would be stuck. Construction: bootstrap {1,2}, add node 3 as a learner and catch it up, then
# promote 3 to voter but isolate it so only {1,2} apply the promotion. With n1 "dead", n2 campaigns
# and asks n3 to vote; n3 (still a learner in its own view) must vote so n2 can win on {2,3}.
